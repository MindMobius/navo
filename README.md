# Navo

一个以“用户评价”为核心的轻量化、社区驱动型导航站。

## 📖 项目简介 (About The Project)

**Navo** 是一个为个人和小型社群设计的导航门户。与追求“大而全”的传统导航站不同，本项目的核心理念在于 **“为什么这个网站有用”** 而非简单地罗列链接。

在这里，每一个被收录的站点都伴随着创建者亲自撰写的 **“使用评价”或“个人印记”**。这使得一个普通的链接，变成了一个带有真实背景、解决特定问题的故事。它不仅仅是一个书签管理器，更是一个记录、分享和发现高质量工具的知识库。

## 🎯 解决的痛点 (The Problem It Solves)

您是否也曾遇到过这些问题？

*   **自建服务管理混乱**：部署了十几个内网服务，每个服务的 IP 和端口都不同，难以记忆和查找。
*   **遗忘工具的核心价值**：收藏了一个很酷的开源项目或在线工具，但几个月后彻底忘了它当初是用来做什么的，以及它最吸引你的点是什么。
*   **传统导航站的信息噪音**：面对成百上-千个分类好的链接，官方描述千篇一律，筛选出真正符合自己当下需求的工具，如同大海捞针。
*   **“宝藏网站”难以沉淀与分享**：发现了一些非常好用但不常用的“神器”，它们静静地躺在收藏夹深处，当朋友遇到同样问题时，你也很难快速地把它们分享出去。

**Navo** 旨在通过一个简单而强大的模式解决以上所有问题。

## 👥 目标受众 (Target Audience)

*   **技术爱好者与开发者**：需要统一管理和备注个人部署的众多服务的用户。
*   **小型团队与社群**：希望在圈子内部共享实用工具集，并附上成员使用心得的群体。
*   **高质量信息筛选者**：不满足于通用导航，希望构建一个属于自己的、小而精的个性化网站库的个人。

## ✨ 功能特性 (Features)

### 1. 核心站点管理 (Core Site Management)
-   [ ] **卡片式布局**：所有站点以信息卡片的形式直观展示。
-   [ ] **灵活的地址支持**：完美支持 `域名`、`IP地址` 以及 `非标准端口号` 的链接。
-   [ ] **API动态更新 (Dynamic Updates via API)**：提供 API 接口，允许用户通过脚本等方式程序化地更新站点地址。这对于家庭宽带等动态 IP 或使用内网穿透的用户极其友好。

### 2. 核心特色：评价体系 (The "Mark" System)
-   [ ] **个人化评价 (The "Mark")**：为每个站点添加一段核心评价，描述“我用它做什么”或“它哪里最酷”。这是每个卡片最核心的展示内容。
-   [ ] **评价快速预览**：在主界面通过悬停等交互，无需点击即可快速查看该站点的核心评价。

### 3. 多用户与权限系统 (Multi-user & Permissions System)
-   [ ] **多用户支持**：支持用户注册和登录，每个人都拥有独立的管理空间。
-   [ ] **双重隐私模式 (Private / Public)**：
    -   **私有 (Private)**：站点默认仅自己可见，非常适合记录内网服务或个人私密链接。
    -   **公开 (Public)**：可一键将站点分享到公共区域，让圈子内所有成员都能看到。

### 4. 社区与聚合 (Community & Aggregation)
-   [ ] **同类服务聚合 (Similar Service Aggregation)**：针对 Emby, Alist, BT面板等流行的自建服务，系统可以将不同用户的公开实例聚合在一起。这方便社群成员发现并访问彼此搭建的同类型服务。
-   [ ] **社区评价聚合 (Community Review Aggregation)**：当多个用户都公开了同一个网站时，系统会自动聚合所有用户对该网站的不同评价，形成一个多角度、更全面的“众评”视图。

### 5. 内容发现与详情 (Content Discovery & Details)
-   [ ] **站点详情页**：点击卡片可进入一个包含更丰富信息的独立页面。
-   [ ] **扩展描述区 (README-like)**：在详情页中，提供一个更详细的区域，用于存放官方介绍或更全面的说明文档（可考虑由AI辅助生成）。
-   [ ] **强大的关键词搜索**：
    -   不仅能搜索站点标题，更能 **深入搜索“用户评价”的内容**。
    -   *例如：搜索“抠图”，就能立即找到那个被用户评价为“抠图神器”的网站。*
